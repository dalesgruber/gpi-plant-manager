{# Shared CSS for the top-center "Saved" toast used by skills, staffing, and settings. Why CSS only (no shared JS)? The three pages use the same toast widget but their autosave control flow diverges enough that sharing the JS would mean bending the API. Specifically: - skills.html — one form, fire-and-forget autosave to form.action - staffing.html — one form, appends ?auto=1, has a "viewing-posted" read-only lock, and runs reapplyVisualState() to re-sync picker pills/summaries after each apply - settings.html — two forms behind an attachAutosaver factory + a single page-header Undo/Redo dispatcher Each page's JS already names its functions consistently (serializeForm, applyState, showSavedToast). Extracting the JS would force one shape onto all three. The CSS, by contrast, is byte-identical across all three pages and is the real win to dedupe. The per-page Undo/Redo button styling (.toolbar, .title-bar, header .page-actions) varies in container selector and is left in the pages. Usage: {% include "_save_toast.html" %} inside the page's
